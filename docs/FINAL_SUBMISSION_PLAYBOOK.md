# Final Submission Playbook

## Objective
Submit a Track 2 package that is complete, aligned, and evidence-driven.

## Step-by-step
1. Run preflight:
   - python scripts/submission_preflight.py
2. Capture Maestro evidence using docs/MAESTRO_EVIDENCE_CAPTURE.md
3. Finalize docs/DEVPOST_SUBMISSION.md links (video and deck)
4. Ensure README links and try-it-out entries are current
5. Verify repository is public and accessible
6. Submit Devpost with Track 2 selected

## Hard rules for Track 2 wording
- Describe process as predictable BPMN with explicit gateways
- Highlight humans, agents, APIs, and orchestration handoffs
- Avoid wording that implies unbounded case evolution

## Last-mile QA
- Test suite passes
- Health endpoints return ok
- Key screenshots are readable and labelled
- Devpost text and README tell the same story

## Confidence checklist
- Technical stack runs end-to-end
- Maestro evidence shows orchestration control plane
- Coding agents bonus proof is visible
- Submission links are valid
