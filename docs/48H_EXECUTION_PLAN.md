# 48-Hour Execution Plan

## Day 1 (Build and validate)
### 0-2h
- Finalize scope: one complete emergency procurement MVP with exception branch
- Set up repo, README, licensing, and submission docs

### 2-6h
- Build or import Maestro BPMN process skeleton
- Wire API tasks to exception and collections agent endpoints
- Configure key variables and gateways

### 6-10h
- Implement human tasks for finance and collections escalations
- Add retries, error routing, and fallback queues
- Run happy path and exception path tests

### 10-14h
- Capture screenshots, logs, and evidence of handoffs
- Draft architecture diagram and value metrics

## Day 2 (Polish and submit)
### 0-3h
- Record 5-minute demo with one happy path and one exception path
- Trim and upload video

### 3-6h
- Finalize Devpost content and project narrative
- Finalize deck with problem, architecture, flow, and outcomes

### 6-8h
- Final QA pass on repo and docs
- Verify public GitHub visibility and links
- Submit before deadline buffer (at least 2 hours early)

## Non-negotiables
- UiPath Automation Cloud visible in demo
- End-to-end orchestration shown in real run
- Human-in-the-loop checkpoint shown clearly
- Public repo includes setup and architecture details

## Solo-team focus
- Timebox tasks tightly and avoid feature creep after first complete run
- Prioritize evidence capture (screenshots/logs) immediately after each passing run
- Keep one fallback demo path pre-staged in case of live run failure
